How a network learns · 1 of 4

“Learning” = making the loss small

The network makes a prediction; we compare it to the right answer; the gap is the loss — one number for “how wrong.” Training is the search for weights that make that number small.

📈 Common losses: cross-entropy for classification (“which class?”), mean-squared-error for numbers (“how much?”). You pick one; the framework computes it for you.

How to fight overfitting

📊 More data

The best cure — harder to memorise a bigger, more varied dataset. Augmentation helps too.

⏹ Early stopping

Stop training at the bottom of the validation curve, before it turns up.

🎲 Dropout

Randomly switch off neurons each step so the net can’t lean on any single path.

⚖️ Regularisation

Penalise large weights (“weight decay”) to keep the model smooth & general.

✂️ Simpler model

Fewer layers/neurons — less capacity to memorise noise.

🔎 Always validate

Judge on held-out data, never the training set. The validation curve is your dashboard.

It all fits in a few lines

The same tiny network — Keras & PyTorch

# Keras — describe the model, .fit does the rest from tensorflow.keras import layers, models model = models.Sequential([ layers.Dense(128, activation="relu"), # hidden layers.Dropout(0.2), # anti-overfit layers.Dense(10, activation="softmax"), # output ]) model.compile(optimizer="adam", loss="sparse_categorical_crossentropy", metrics=["accuracy"]) model.fit(x_train, y_train, epochs=5, batch_size=32, validation_split=0.1) # Epoch 5/5 loss: 0.07 accuracy: 0.98
# PyTorch — you can see the training loop import torch, torch.nn as nn model = nn.Sequential( nn.Linear(784, 128), nn.ReLU(), nn.Dropout(0.2), nn.Linear(128, 10), ) loss_fn = nn.CrossEntropyLoss() opt = torch.optim.Adam(model.parameters(), lr=1e-3) for epoch in range(5): # the training loop opt.zero_grad() loss = loss_fn(model(xb), yb) # forward + loss loss.backward() # backprop opt.step() # gradient descent

👉 Every word here is a concept from this module: Dense/Linear layers, ReLU, Dropout, a loss, an optimiser (Adam = smarter gradient descent), epochs, backward() = backprop.

Recap

Five things to carry forward

1 · A neuron is tiny

Weighted sum + bias + activation. A network is just many of them in layers.

2 · Activations give power

Non-linearity (ReLU, sigmoid) lets a network model any pattern.

3 · Learning = shrinking loss

Gradient descent walks the weights downhill on the loss; the learning rate is the step size.

4 · Backprop is automatic

It assigns blame backwards; the framework computes it — you call .backward().

5 · Watch the validation curve

Training loss always falls; when validation turns up, you’re overfitting. Stop, regularise, or get more data.
