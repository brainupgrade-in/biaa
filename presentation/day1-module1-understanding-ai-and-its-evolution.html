Setting the scene

Why is everyone talking about AI now?

AI is not new — the field is ~70 years old. What changed is that three long-building forces finally converged, tipping AI from research curiosity to everyday tool.

📊 Data

The internet, sensors & digitisation produced oceans of training data — text, images, code, transactions.

⚡ Compute

GPUs (built for games) turned out to be perfect for training neural networks — orders of magnitude cheaper, faster, parallel.

🧮 Algorithms

Breakthroughs — deep learning (2012), then the transformer (2017) — let models scale and actually use all that data & compute.

💡 Data + Compute + Algorithms. Remove any one and the modern AI wave doesn’t happen. This is why a 70-year-old idea feels brand new — and why understanding the evolution helps you use today’s tools well.

The landscape · 1 of 5

What is Artificial Intelligence?

  • A working definition: getting computers to perform tasks that normally require human intelligence — perceiving, reasoning, deciding, understanding language.
  • The key shift: from writing explicit rules to letting machines learn patterns from examples.
  • It’s a spectrum, not a switch — from a spam filter to a self-driving car to a conversational assistant.
  • Already everywhere — you rely on it dozens of times a day without naming it.

The shift, in code: writing rules vs. learning them

# ✖ The OLD way — hand-written rules # "Is this image a handwritten 8?" def is_eight(pixels): if has_two_loops(pixels) and \ loops_stacked(pixels): return "8" # ...what about a slanted 8? # ...a thick pen? a wobbly loop? # thousands of edge cases → hopeless
# ✔ The ML way — learn from examples from tensorflow.keras import models, layers model = models.Sequential([ layers.Dense(128, activation="relu"), layers.Dense(10, activation="softmax"), ]) # show it labelled digits → it learns model.fit(images, labels, epochs=5) # accuracy: 0.97 🎉 no rules written

💡 You never tell the model what an 8 looks like — you show it examples and it learns the pattern. That handful of lines is your Lab 1.1 this afternoon (MNIST digit classifier).

The evolution · what ended the winter

What finally made deep learning work

📊 Big labelled data

ImageNet (14M labelled images) and the web gave models enough examples to learn from.

⚡ GPU compute

Graphics chips train neural nets in parallel — what took weeks now took hours.

🧮 Better algorithms

Deeper networks + backpropagation at scale, then the transformer in 2017.

📈 Scaling laws: with deep learning, capability tends to grow predictably as you add data, compute & model size. That single observation drove the last decade — and the rise of giant pre-trained “foundation models” (GPT, BERT) you’ll use on Day 2.

…but scaling isn’t free or infinite — cost, data limits & reliability still bite. We’ll be honest about that later in this module.

Three ways machines learn

🎯 Supervised

Learn from labelled examples — input + the right answer.

  • Image → “cat” / digit → “8”
  • Review → “positive”

→ Lab today (MNIST) & BERT, Day 2

🔗 Unsupervised

Find structure with no labels — group & relate.

  • Cluster similar customers
  • Embeddings behind search / RAG

→ Underlies agent memory & retrieval

🎮 Reinforcement

Learn by trial, reward & error toward a goal.

  • Game-playing, robotics
  • RLHF — made ChatGPT helpful

→ Why modern assistants feel aligned

🧠 Most of this course rests on supervised learning + huge pre-trained models. You’ll use all three ideas without needing the math — intuition first, code alongside.

Generative AI · the creative leap

Generative AI & Large Language Models

A Large Language Model (LLM) is a deep neural network trained to predict the next word over vast text. At scale, that simple goal yields models that read, reason, summarise, draft and code.

✅ Where LLMs shine

  • Reading & summarising long documents
  • Explaining concepts in plain language
  • Drafting emails, reports, code
  • Reasoning step-by-step over what you give it

⚠️ Where a bare LLM falls short

  • Hallucination — confident, wrong answers
  • No live data — frozen at its training cut-off
  • It only talks — can’t fetch, calculate or act
  • No memory of your world by default

🔑 An LLM is a brilliant analyst with no phone, no data feed, and no hands — it can only tell you things. Give it tools and a goal, and it becomes an agent. That’s the next — and final — step.

The honest view

What today’s AI is — and isn’t — good at

💪 Genuine strengths

  • Scale & speed — millions of items, tirelessly
  • Consistency — no fatigue, no mood
  • Language fluency — read, summarise, draft
  • Pattern-finding beyond human bandwidth

⚠️ Real limits & risks

  • Hallucination — fluent but wrong
  • Bias — inherits it from training data
  • No true understanding — pattern, not meaning
  • Opaque & data-hungry — hard to fully explain

🤝 The professional stance, all week: use AI for leverage, keep a human in the loop, and verify outputs. We return to this in depth in Day 5 — Ethics & Responsible AI.

Vocabulary you’ll hear all week

Key terms, demystified

TermIn one plain sentence
ModelA trained program that maps inputs to outputs (e.g. image → label).
TrainingThe process of adjusting a model on data so it makes fewer errors.
Parameters / weightsThe numbers inside a model that training tunes; modern LLMs have billions.
Neural networkA model of layered units loosely inspired by the brain; “deep” = many layers.
TokenA chunk of text (~¾ of a word) an LLM reads & predicts.
InferenceUsing a trained model to make a prediction (vs. training it).
Pre-trained / fine-tuneA model trained on broad data, then specialised on your task (Day 2).
PromptThe instruction you give a generative model.
AgentAn LLM given tools, a goal & a loop so it can act, not just answer.

BReinforcement learning

AUnsupervised learning
BReinforcement learning
CSupervised learning ✓
DGenerative modelling

WHY  Supervised learning uses labelled pairs. Unsupervised finds structure in unlabelled data; reinforcement learns from trial-and-error rewards.

AIt always runs faster
BIt needs no data at all
CIt can never make a mistake
DIt learns patterns from data instead of following hand-written rules

Learn the rules from data

Rules written by hand vs. rules learned from examples.

AIt always runs faster
BIt needs no data at all
CIt can never make a mistake
DIt learns patterns from data instead of following hand-written rules ✓

WHY  Traditional programming: you write the rules. Machine learning: the model learns the rules from data — the shift that makes modern AI possible.

Quick check · Q9 / 10

A key limitation of today’s AI to keep in mind is that it…

ATruly understands meaning like a human
BIs always objective and unbiased

Answer · Q9 / 10

Confidently wrong

Why a human stays in the loop.

ATruly understands meaning like a human
BIs always objective and unbiased
CCan be confidently wrong and mirrors biases in its training data ✓
DNever needs human oversight

WHY  AI pattern-matches without real understanding: it can be confidently wrong (hallucinate) and inherits bias from its data. That is why humans stay in the loop — the theme of Day 5.

Quick check · Q10 / 10

How does an AI agent differ from a plain model?

AIt is simply a larger neural network
BIt can plan, use tools, and take actions toward a goal — not just return one output
CIt removes the need for a model underneath
DIt can only classify images
Answer · Q10 / 10

An agent acts

Model = answer; agent = pursue a goal.

AIt is simply a larger neural network
BIt can plan, use tools, and take actions toward a goal — not just return one output ✓
CIt removes the need for a model underneath
DIt can only classify images

WHY  A model maps input to output once. An agent wraps a model in a loop — it plans, calls tools, observes, and acts toward a goal. That is the rest of this course.
