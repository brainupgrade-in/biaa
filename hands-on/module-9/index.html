Day 5 · Module 9 — Hands-on Labs

Agents in Finance, Healthcare & Cybersecurity — build the financial-report insight agent
12 labs | 6 Beginner · 3 Intermediate · 3 Advanced | ~5h 40m total | Near-real: build → run for real → your turn
How these labs work (near-real). Each notebook follows the same rhythm — read the Concept, fill the real ___ blanks in Build it (the real grounding / citation / compute logic, or the real create_agent wiring), then Run it and read the output — and, for the agent labs, the real message trace. Finish with an open Your turn. There is no auto-grader. Each lab writes to its own /tmp/biaa-lab-09-NN/ working dir. Stuck? Every lab has a full answer key under solutions/ — try first, then check.
Setup. These labs use the real LangChain 1.x (langchain, langchain-core, langchain-groq — all in the course biaa-venv). You build the financial-report insight agent (the client’s Lab 5.1) piece by piece: ground & cite every figure, compute derived metrics, flag anomalies, the no-advice guardrail, withholding the trade tool, grounding validation, the audit trail, privacy/redaction, and assistive-not-autonomous. The agent-assembly labs (10–12) bind real @tool functions with create_agent and drive a real hosted model (ChatGroq("openai/gpt-oss-20b"), key in .env as GROQ_API_KEY, free at console.groq.com) — you read the real trace where the agent grounds & cites a figure. If the key is unset, the “Run it for real” cells print how to set it instead of crashing. Financial math uses a small AST-based safe evaluator (never bare eval); a @tool always catches its own errors and returns a string.
🛡️ Responsible-AI note. The insight agent is informational only. It grounds & cites every figure, gives no investment advice, and has no trade tool — a human analyst reviews and decides. That guardrail (assistive, not autonomous) is the point of the module, not a limitation.
