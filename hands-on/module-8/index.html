Day 4 · Module 8 — Hands-on Labs

Multi-Agent Collaboration & Decision Making — build the customer-service chatbot
12 labs | 6 Beginner · 3 Intermediate · 3 Advanced | ~5h 40m total | Experiential: concept → demo → practice
How these labs work. Each notebook follows the same rhythm — read the Concept, run the Demo, then complete Your Turn by replacing every ___ placeholder. Run the grader cell at the bottom: it prints [PASS] / [FAIL] / [TODO] and a final Score. Each lab writes to its own /tmp/biaa-lab-08-NN/ working dir. Stuck? Every lab has a full answer key under solutions/ — try first, then check.
Setup. These labs use the real LangChain (langchain, langchain-core, langchain-ollama — all in the course biaa-venv). You build the multi-agent customer-service chatbot (the client’s Lab 4.2) piece by piece: specialist roles, the supervisor/router, shared state, the orchestration shapes (pipeline, fan-out, handoff), the decision mechanisms (voting, critique, synthesis), observability, and the assembled chatbot — whose specialists are real create_agent agents. The graded cells assert only on the deterministic parts you build (routing, synthesis, the refund gate, agent structure) and never call an LLM, so they verify offline with no API keys and no network. Cells marked Optional — run it for real call a live local model (ollama run llama3.2:1b, or ChatGroq with a GROQ_API_KEY) and self-skip if none is reachable.
