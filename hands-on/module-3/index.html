Day 2 · Module 3 — Hands-on Labs

Why Transformers? — tokenize, embed, attend & generate
12 labs | 6 Beginner · 3 Intermediate · 3 Advanced | ~6h 15m total | Near-real: build → run real models → observe
How these labs work (near-real). Each notebook runs real Hugging Face Transformers locally on CPU. Read the Concept, fill the real ___ blanks in Build it (real tokenizer / model / decoding calls), Run it for real to see the actual model output, note What to notice, then finish with an open Your turn. There is no auto-grader — the goal is real model output you can read. Each lab writes to its own /tmp/biaa-lab-03-NN/ working dir. Every lab has a full answer key under solutions/ — try first, then check.
Setup. Use the course biaa-venv (has transformers, torch, scikit-learn, matplotlib, langchain-groq). Labs load small CPU-friendly models from the HF hub — distilbert-base-uncased, all-MiniLM-L6-v2, prajjwal1/bert-tiny, distilgpt2; the first use of each downloads the weights (needs network), then they are cached. The hosted “GPT API” cell in Lab 3.10 uses ChatGroq (GROQ_API_KEY in .env). If a blank is unfilled or the first download has no network, cells print a friendly note instead of crashing.
