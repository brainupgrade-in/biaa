Day 4 · Module 7 — Hands-on Labs

Task Automation with AI Agents — build the email-drafting agent
12 labs | 6 Beginner · 3 Intermediate · 3 Advanced | ~5h 40m total | Experiential: concept → demo → practice
How these labs work. Each notebook follows the same rhythm — read the Concept, run the Demo, then complete Your Turn by replacing every ___ placeholder. Run the grader cell at the bottom: it prints [PASS] / [FAIL] / [TODO] and a final Score. Each lab writes to its own /tmp/biaa-lab-07-NN/ working dir. Stuck? Every lab has a full answer key under solutions/ — try first, then check.
Setup (almost none). Just Python 3.10+ — every graded step uses only the standard library. You build the email-drafting agent (the client’s Lab 4.1) piece by piece: the automation pipeline, the three patterns (draft · extract · route), structured output, validation, retries & idempotency, and the draft-not-send guardrail. The agent-assembly labs reuse the LangChain-shaped shim from Module 6 (same names & shapes), driven by a deterministic scripted model — so there are no API keys, no model downloads, no network. Each Advanced lab ends with an optional, non-graded cell that runs the real LangChain (pip install langchain langchain-ollama + ollama run llama3.2:1b, or ChatGroq with a GROQ_API_KEY); it skips cleanly if absent.
